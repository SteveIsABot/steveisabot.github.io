Infinite Terrain Generation

Stage 1 - Creation of Terrain Grids

Using a list of vertices and a list of triangles to create a simple grid of varying size. The terrain is generated via a perlin noise algorthim to generate the height value for each vertex creating a bumpy surface. The algorthim uses the world position of each vertex as parameters so each grid won't have gaps in between each other and look connected when placed together.

Stage 2 - Placing of terrain grid

Assuming there are no other grids in the world the first terrain grid is always placed underneth the camera. From that grid we find the next potential place where I can place another terrain grid. The next potential place for a grid to spawn needs to be in the camera view, and next to another terrain grid. If there is terrain grids that already exsit in world then the algorthim searches for the furtherest away terrain grid. From that terrain grid it trys to find other potential spawn points for new terrain grids.

Stage 3 - Applying colour to terrain grid

The terrain grid sends a colour gradient as a texture to be sampled from. It also receives the lowest and highest hight of the terrain to create a height percentage for each vertex. The shader samples the texture (The colour gradient) based on it's height percentage in the world creating a colour blend for the terrain grid. Each terrain grid has the same gradient applied to it since the all terrain grids will have varying vertices heights.

Stage 4 - Applying texture to terrain grid

The terrain grid sends 4 textures, a list of start heights for each texture, scale for each texture and bledning for each texture to the shader. These textures are applied based of the height percentage of each vertex and scale of for texture. Since there are 4 textures being applied each vertex is checked 4 times to see apply each texture correctly. The shader logic was primarly based of this video: Procedural Landmass Generation by Sebastian Lague
